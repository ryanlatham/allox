# Codex Project Notes

This project was generated by `allox` for a Codex-led workflow.

- Opening the repo in Codex is enough for the normal development path.
- Codex should pick up `AGENTS.md` automatically and use the installed `allox` runtime when internal orchestration is needed.
- Hidden repo-local files under `.allox/scripts/` are compatibility shims around the installed runtime, not the primary user workflow.
- Worktree setup helpers live behind `allox project worktree-setup`.
- Inspectable workflow configuration, prompts, schemas, and templates live under `allox/`.
- Shareable project-local Codex environment files belong under `.codex/` when needed.

If you need to inspect the environment manually, use `allox doctor` from an installed tool or `uv run allox doctor` from a local checkout.
